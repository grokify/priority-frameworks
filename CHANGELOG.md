## [v0.1.0] - 2026-06-06 ([`32286b3`](https://github.com/grokify/priority-frameworks/commit/32286b3))

### Highlights

- Go library for standardized priority level definitions with 5 built-in frameworks and cross-framework normalization

### Added

- Core `Framework` and `Level` types for defining priority systems ([`b181fd0`](https://github.com/grokify/priority-frameworks/commit/b181fd0))
- Built-in `Severity` framework (Critical, High, Medium, Low, Info) ([`57ffbb9`](https://github.com/grokify/priority-frameworks/commit/57ffbb9))
- Built-in `Priority` framework (P0-P4 urgency levels) ([`57ffbb9`](https://github.com/grokify/priority-frameworks/commit/57ffbb9))
- Built-in `IETF` framework (RFC 2119 requirement levels: MUST, SHOULD, MAY) ([`57ffbb9`](https://github.com/grokify/priority-frameworks/commit/57ffbb9))
- Built-in `MoSCoW` framework (Must Have, Should Have, Could Have, Won't Have) ([`57ffbb9`](https://github.com/grokify/priority-frameworks/commit/57ffbb9))
- Built-in `General` framework (High, Medium, Low) ([`57ffbb9`](https://github.com/grokify/priority-frameworks/commit/57ffbb9))
- `Normalize()` function for converting levels to comparable numeric scale (0-100) ([`7c598cd`](https://github.com/grokify/priority-frameworks/commit/7c598cd))
- `CompareAcross()` function for comparing priorities across different frameworks ([`7c598cd`](https://github.com/grokify/priority-frameworks/commit/7c598cd))
- `MapTo()` function for finding equivalent levels between frameworks ([`7c598cd`](https://github.com/grokify/priority-frameworks/commit/7c598cd))
- `GetLevelByID()` and `GetLevelByNameOrAlias()` lookup functions ([`b181fd0`](https://github.com/grokify/priority-frameworks/commit/b181fd0))
